# Phase 36 Implementation Plan — Public-Playtest Visual Finish Pass

**Authorization:** Owner requested a more finished visual presentation suitable for initial public testing on 2026-07-23.

## Goal

Elevate the established Wayroot route from prototype-like runtime primitives toward a cohesive, polished alpha presentation using original source-controlled/runtime-composed art only.

## Visual focus

- Establish a coherent final-alpha palette, lighting hierarchy, terrain/material variation, depth fog, and landmark silhouettes across Meadow, shelter, Grove, Glade, and Bloomwell.
- Improve player, Mossling, slime, Guardian, merchant, gathering nodes, and Iron Edge readability with layered silhouette/detail treatment at mobile scale.
- Upgrade HUD/control presentation from debug-like blocks toward a consistent quiet fantasy UI treatment while preserving safe-area and semantic controls.
- Improve camera framing/composition and route staging without changing collision, target positions, progression, or controls.
- Add restrained finishing effects only through existing URP-safe property-block/material paths and mobile safeguard/accessibility gates.

## Non-goals

- No external/generated/copyrighted assets, asset packs, shaders, music, fonts, maps, regions, quests, items, currencies, progression, combat values, save schema, or platform release claim.
- This is an **initial public-playtest visual finish**, not App Store readiness: physical iPhone, Mac/Xcode, signing, privacy/release identity, and external playtest evidence remain separate gates.

## Acceptance

- Actual Windows player screenshots show a cohesive finished-alpha look: no magenta materials, oversized/hovering geometry, placeholder-like obstruction, duplicate actors, or control/HUD occlusion.
- Full route remains playable and readable at desktop/mobile target aspect.
- Compile, tests, review build/smoke, actual multi-state visual capture, documented public-playtest limitations, clean commit/push pass.
